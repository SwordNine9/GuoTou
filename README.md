# 国投电力政策资讯专刊生成系统（GuoTou）

基于 **Streamlit + python-docx + DeepSeek API** 的自动化报告生成工具。  
它可以将结构化政策数据（JSON）与 Word 模板合并，自动完成：

- 指标占位符替换（`**`）
- 装机结构饼图绘制并插入
- 政策条目按章节批量写入
- （可选）调用 DeepSeek 生成“政策解析”
- 支持上传 txt/md 资料做轻量 RAG 检索增强
- 一键导出可下载的 `.docx` 报告

---

## 1. 项目结构

```text
.
├── app.py                      # Streamlit 主程序
├── api.py                      # FastAPI 接口入口（非 Streamlit 前端可对接）
├── policy_data.json            # 示例输入数据
├── 国投电力业务发展部政策资讯专刊（模板）.docx  # 示例模板
├── map.py / map2.py            # 早期脚本与实验代码
├── section_mapping.json         # 章节映射配置（可自定义）
├── logs/app.log                 # 运行日志（自动生成）
├── generation_history.jsonl     # 生成历史（自动生成）
└── README.md
```

> 当前推荐使用 `app.py` 启动 Web 界面进行操作。

---

## 2. 核心功能

- **模板占位符替换**  
  按顺序将 JSON 中的关键指标填入模板中的 `**` 占位符。
- **自动生成饼图**  
  使用 `hydro/thermal/nuclear/solar/wind` 装机容量绘制饼图，并插入“（一）全国电力供应数据”段落后。
- **政策内容自动排版写入**  
  根据模板章节标题（如“二、政府文件”“一、电力交易新政”）定位并插入政策内容。
- **统一字体与版式**  
  插入内容统一设置为仿宋，标准段落字号为三号（16pt）。
- **AI 政策解析（可选）**  
  填入 DeepSeek API Key 后，系统为每条政策摘要自动生成简要解析。

- **RAG 增强解析（新增）**  
  在侧边栏开启“启用 RAG 增强解析”，上传 `txt/md` 资料后，系统会自动切分文本并按关键词重叠检索相关片段，拼接到 AI 提示词中提升解析贴合度。

## 3. 运行环境

- Python 3.9+（建议 3.10/3.11）
- 可安装以下依赖：
  - `streamlit`
  - `python-docx`
  - `matplotlib`
  - `openai`
  - `fastapi`
  - `uvicorn`
  - `python-multipart`

安装示例：

```bash
pip install streamlit python-docx matplotlib openai fastapi uvicorn python-multipart
```

---

## 4. 快速开始

### 4.1 启动应用

在项目根目录执行：

```bash
streamlit run app.py
```

浏览器打开后：

1. （可选）在左侧输入 `DeepSeek API Key`
2. （可选）开启“启用 RAG 增强解析”并上传 `txt/md` 知识库资料
3. 上传 `policy_data.json`
4. 上传 Word 模板（`.docx`）
5. 点击“🚀 开始生成报告”
6. 下载生成的 Word 文件


### 4.2 不使用 Streamlit：FastAPI 接口模式（推荐前后端分离）

如果你不想使用 Streamlit，可直接启动 API 服务：

```bash
uvicorn api:app --host 0.0.0.0 --port 8000 --reload
```

可用接口：
- `GET /health`：健康检查
- `POST /validate`：校验 JSON + 模板
- `POST /generate`：生成 `.docx` 文件（支持 RAG 文件上传）

这样你可以用 Vue/React/Angular 或企业内部前端系统对接该 API。

## 5. 输入数据说明（JSON）

`policy_data.json` 顶层建议包含以下字段：

- `important_data_values`：关键指标（用于替换 `**` 与绘图）
- `gov_policies`：政府文件列表
- `energy_new_policies`：能源新政（按子章节分类）
  - `电力交易新政`
  - `区域电价政策`
  - `重点开发政策`
- `references`：参考资料列表

示例（节选）：

```json
{
  "important_data_values": {
    "report_date": "2025年8月底",
    "total_capacity": "36.9",
    "hydro_capacity": "12.0",
    "thermal_capacity": "15.6",
    "nuclear_capacity": "2.8",
    "solar_capacity": "10.5",
    "wind_capacity": "8.1"
  },
  "gov_policies": [
    {
      "title": "电力市场计量结算基本规则",
      "agency": "国家发展改革委、国家能源局",
      "date": "2025年07月18日",
      "summary": "..."
    }
  ]
}
```

## 6. 模板要求（非常重要）

程序通过“**章节标题关键字匹配**”定位插入点。请确保模板包含以下标题文本（至少关键字一致）：

- `（一）全国电力供应数据`（用于插入饼图）
- `二、政府文件`
- `一、电力交易新政`
- `二、区域电价政策`
- `三、重点开发政策`
- `参考资料`

如果模板中缺少对应标题，系统会跳过该部分并在界面给出提示。

---

## 7. RAG 使用说明

- 在侧边栏打开 **启用 RAG 增强解析**。
- 上传一个或多个 `txt` / `md` 文件作为知识库。
- 使用“RAG 检索片段数”控制每条政策注入模型的参考片段数量。
- 当前实现已升级为**轻量向量检索（TF-IDF + Cosine）**，无需额外向量数据库即可获得更稳定的相关性排序；后续可继续替换为 FAISS/Milvus。

---

## 8. DeepSeek API 说明

- 应用通过 `OpenAI` SDK 并指定 `base_url=https://api.deepseek.com` 调用模型。
- 当前使用模型：`deepseek-chat`。
- 如果未填写 API Key，系统仍可生成报告，但“政策解析”将使用占位提示文本。

## 8.1 已落地的扩展能力（本版本）

- **前端指标看板**：页面新增最近任务数、成功率、平均耗时、当前 RAG 文本块数等核心指标。

- **配置化章节映射**：支持通过 `section_mapping.json` 自定义“模板标题 -> JSON路径 -> 内容类型”映射。
- **模板预检查**：生成前自动检查章节标题缺失、图表锚点缺失、占位符数量不足。

- **结构化日志**：运行日志写入 `logs/app.log`。
- **生成历史追踪**：每次生成记录到 `generation_history.jsonl`，侧边栏可查看最近记录。
- **RAG 可追溯输出（可选）**：支持把检索片段写入报告正文，便于审阅。

## 9. 常见问题

### Q1：点击生成后提示某章节未找到
请检查模板标题是否被修改（包括空格、标点、序号）。建议保持与默认模板一致。

### Q2：中文图表标题乱码
当前图表标题已使用英文以降低字体兼容风险。如需中文标题，可自行配置 Matplotlib 中文字体。

### Q3：API 报错导致解析失败
确认 API Key 有效、网络可访问 DeepSeek 服务，并检查账户权限与额度。

## 10. 后续可扩展方向

为了让系统从“可用”走向“稳定可运营”，建议按以下方向分阶段演进：

### 10.1 检索与生成（RAG）能力升级
- 从当前关键词检索升级为**向量检索**（如 FAISS / Milvus / pgvector）。
- 引入重排（rerank）模型，提升召回片段与政策摘要的相关性。
- 支持多源知识库（政策原文、公司制度、历史专刊）并进行权限隔离。
- 为每条“政策解析”增加可追溯引用（来源文件 + 段落定位）。

### 10.2 模板系统工程化
- 将章节关键字映射抽离为配置文件（YAML/JSON），支持不同业务条线复用。
- 支持多模板版本管理（模板版本号 + 兼容数据结构声明）。
- 增加模板预检查：一键检测缺失标题、占位符数量不匹配、样式异常。

### 10.3 输出与排版能力增强
- 增加 PDF 导出链路（docx -> pdf）并提供版式一致性校验。
- 支持图表主题配置（配色、字号、标题语言）及更多图表类型（柱状/趋势）。
- 支持自动目录、页眉页脚、页码、批注与修订模式导出。

### 10.4 质量保障与可观测性
- 增加单元测试、集成测试与样例回归测试（固定 JSON + 模板对比输出）。
- 引入结构化日志与错误分级，便于定位“模板错误 / 数据错误 / API 错误”。
- 增加指标看板（生成耗时、失败率、RAG 命中率、API 成本）。

### 10.5 安全与生产化
- API Key 迁移到密钥管理服务（环境变量/密管平台），避免明文输入和持久化风险。
- 增加用户身份认证与操作审计（谁在何时基于何数据生成了报告）。
- 增加数据合规策略（敏感字段脱敏、留存周期、下载权限控制）。

### 10.6 使用体验与自动化
- 增加“任务中心”与异步队列，支持批量生成和生成历史追踪。
- 支持定时任务：按周/月自动拉取数据并生成专刊草稿。
- 支持“人审工作流”：AI 草稿 -> 人工修订 -> 定稿归档。
